# W1.5-01: Audit silent drop on I/O failure

## Intent
`infrastructure/audit.py:190` catches I/O exceptions on audit event writes
and silently drops them. The audit log is hash-chained — a silently dropped
event breaks chain integrity without surfacing any error. This is a protocol
guarantee, not a cosmetic issue.

## What to change
When an audit write fails, raise AuditError (or an appropriate existing
exception type — check the codebase first). Do not swallow the exception.
The caller is responsible for deciding whether to halt or continue.

If no AuditError type exists, create it in core/interfaces.py.

## Acceptance criteria
- I/O failure on audit write raises, does not silently return
- Existing callers of the audit write path are updated to handle or
  propagate the exception appropriately
- No silent drops anywhere in audit.py

## Testing
- Unit test: mock I/O failure on audit write, assert exception raised
- Unit test: assert no bare except / silent return in the write path
- Existing audit test suite passes clean

## Constraints
- Do not change the hash-chain logic
- Do not change the audit event schema
- Touch only audit.py and any callers that need updating
