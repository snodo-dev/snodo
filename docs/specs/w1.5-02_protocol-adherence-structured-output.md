# W1.5-02: protocol_adherence structured output upgrade

## Intent
validators/protocol_adherence.py is running the original free-text LLM
parse pipeline from the initial release. It was never upgraded when
structured output and config-driven tokens landed on llm_validator.
Same failure mode: max_tokens=500 hardcoded, no JSON enforcement,
fragile regex parse, silent warn on failure.

Bring it to parity with llm_validator's single-completion path.

## What to change
Read validators/llm_validator.py first — that's the reference implementation.
protocol_adherence needs the same upgrade:

- supports_response_schema check + _call_llm_structured method
- response_format=ValidatorResult on the structured path
- completion_tokens from config, not hardcoded 500
- Non-greedy regex fix in _try_extract_json:
  from r'```(?:json)?\s*\n?(.*?)```'
  to   r'```(?:json)?\s*\n?([\s\S]*?)```'
- Add "error" to VALID_SEVERITIES (line 27) — parity with llm_validator

## Acceptance criteria
- Structured path used when supports_response_schema returns True
- max_tokens sourced from config, not hardcoded
- _parse_response retained as fallback for unsupported models
- VALID_SEVERITIES includes "error"
- Non-greedy regex fixed

## Testing
- Same four test patterns as llm_validator TestStructuredOutput:
  structured path bypasses parse, markdown prose handled, unsupported
  model falls back, config max_tokens passed
- Existing protocol_adherence test suite passes clean

## Constraints
- llm_validator.py is the reference — match its pattern, don't invent
- Touch only protocol_adherence.py unless a new exception type is needed
