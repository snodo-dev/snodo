# Study conventions

Rules every study must follow.

## Randomness

- **Always** use `_common.seeds.rng()` (or `rng(SEED)`), never bare `np.random`.
- The master seed is `SEED = 42` in `_common/seeds.py`.  Changing it
  regenerates every study.
- Each study must be fully deterministic: same input → same output, always.

## Figures

- All figures use `_common.plotting.apply_paper_style()` before any plot call.
- Save via `save_figure(fig, name)` — emits SVG to the calling study's
  `outputs/` directory.
- SVG is the standard format: vector, paper-ready, diffable.
- Figure styling (fonts, palette, sizing) lives in `_common/plotting.py`.
  If a study needs a one-off, parameterize style rather than copy-pasting.

## Data

- Save via `save_data(df, name)` — emits CSV to `outputs/`.
- CSVs are machine-readable, commit-friendly, and importable by the paper.
- Never hand-edit files in `outputs/` — they are regenerated by `make studies`.

## Outputs directory policy

- `outputs/` is regenerable.  Commit final paper figures; gitignore scratch
  intermediates at your discretion.
- The `.gitignore` at repo root excludes `*.svg` and `*.csv` inside
  `studies/.../outputs/` **except** for `_smoke/outputs/` which is committed
  as a regression guard.  Individual studies may override with an inline
  `.gitignore` if they want committed outputs.

## Study structure

- Each study is a single `notebook.py` marimo notebook in its own directory.
- Self-contained: no imports from other studies or the `snodo` package
  (except through the `_common` helpers).
- Run headless: `python studies/<name>/notebook.py`
- Run interactively: `marimo edit studies/<name>/notebook.py`

## Paper handoff

Figures flow from study `outputs/` into the paper figure directory via a
manual copy (or symlink).  The runner does **not** automate this — it
belongs to the paper build system, not the study runner.

```
cp studies/<name>/outputs/*.svg paper/figures/
```

## Dependency policy

- Study dependencies (`marimo`, `numpy`, `matplotlib`, `pandas`) are listed
  in `studies/requirements.txt` and the `studies` optional extra in
  `pyproject.toml`.
- They must **never** leak into the main `snodo` package install.
- Install with: `pip install -r studies/requirements.txt` or
  `pip install -e ".[studies]"`.
